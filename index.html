📊 มาตรฐาน Barcode Pattern — WFP WMS v2.0

ใช้ Code-128 บาร์โค้ด เนื้อหาคั่นด้วย | (Pipe) เรียง 7 ท่อนคงที่ ทั้ง RM และ FG ใช้รูปแบบเดียวกัน ทำให้ WMS parser อ่านด้วย index เดียว:

0: Part | 1: Lot | 2: Qty | 3: Unit | 4: Rolls | 5: Ref | 6: PO

RM ตัวอย่าง: AAP050C400|L04-R1-2026|250|KGS.|1|RC-1234|PO-2026-001

FG ตัวอย่าง: ABT-P-0001|FG-L02-2026|800|ชิ้น|1||PO-CUST-099

⚠ Ref/PO ที่ไม่มีข้อมูล — เว้นว่างไว้ระหว่าง || (อย่าลบท่อน) เพื่อให้ parser ยังแยก index ได้ถูกต้อง

📋 Google Sheets — Suppliers Sheet Structure

Sheet ID: 1J1iMGwCapKyWiymP3zm0jrEUNWfxMdwCdxN-SbsdziQ → Sheet: Suppliers

A: Barcode (string เต็ม 7 ท่อน)
B: Part No.
C: วัสดุ / ชื่อสินค้า
D: ขนาด (Size)
E: หน่วย (Unit)
F: Supplier
G: ประเภท (RM / FG) ← คอลัมน์ใหม่
H: วันที่สร้าง ← คอลัมน์ใหม่

WMS backend ใช้ lookupBarcode(bc) match คอลัมน์ A → return Part, Name, Size, Unit, Supplier ทั้งหมด

🔗 การเชื่อมต่อ GAS Web App (สำหรับ GitHub Pages)

เนื่องจาก Label Generator host บน GitHub Pages (static) จึงต้องมี GAS Web App เป็น API กลาง:

  • สร้าง Google Apps Script ใหม่ → ใส่โค้ด GAS Endpoint ด้านล่าง → Deploy as Web App (Anyone)
  • คัดลอก Web App URL → แปะที่ค่า GAS_ENDPOINT ในไฟล์ HTML บรรทัดที่ 2
  • ตั้งค่า Sheet ID = 1J1iMGwCapKyWiymP3zm0jrEUNWfxMdwCdxN-SbsdziQ
// GAS Code — doPost() รับ JSON แล้ว append ลง Suppliers sheet
function doPost(e) {
  const d = JSON.parse(e.postData.contents);
  if (d.action === 'appendSupplier') {
    const ss = SpreadsheetApp.openById(d.sheetId);
    ss.getSheetByName(d.sheetName).appendRow(d.row);
    return ContentService.createTextOutput(JSON.stringify({success:true}))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
