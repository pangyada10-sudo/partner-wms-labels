🔗 แหล่งข้อมูลสินค้า (Google Sheets)

วางลิงก์ Published CSV — ข้อมูลจะถูกโหลดและ Cache ไว้
Partner ค้นหาได้แต่ ไม่เห็นรายการทั้งหมด

— ยังไม่ได้ตั้งค่า
📋 รูปแบบ Google Sheets (Column A→E)
A: Part No.  B: Product Name  C: RM หรือ FG  D: Unit  E: Supplier/Customer
แถว 1 = Header (ข้ามอัตโนมัติ) · ตัวอย่าง: ABT-P-0001, ERNI 084315, FG, Pcs, Erni
วิธี Publish Google Sheets เป็น CSV
ไฟล์ → แชร์ → เผยแพร่ไปยังเว็บ → เลือก Sheet → Comma-separated values (.csv) → เผยแพร่ → คัดลอก URL

📂 อัปโหลดไฟล์ CSV โดยตรง แนะนำถ้าโหลดจาก URL ไม่ได้

หากกด "โหลดข้อมูล" ด้านบนแล้วไม่สำเร็จ (CORS Proxy เข้าไม่ถึง) — ให้ ดาวน์โหลด URL ด้านบนในเบราว์เซอร์ (จะได้ไฟล์ .csv ลงเครื่อง) แล้วอัปโหลดไฟล์นั้นที่นี่แทน
วิธีนี้ ไม่ต้องพึ่ง Proxy เลย เชื่อถือได้ 100%

📋 Print Queue — วิธีใช้งาน

1. เลือก รูปแบบการพิมพ์: กระดาษ A4 หรือ Thermal Roll
2. เลือก ขนาด Label: Mini หรือ Sticker 100×50mm
3. กรอกข้อมูล Label ทางซ้าย
4. กด ➕ เพิ่มลงรายการพิมพ์ — สามารถเพิ่มได้เรื่อยๆ
5. กด 🖨️ พิมพ์ทั้งหมด

🖨️ รูปแบบการพิมพ์ที่รองรับ (v6.4)

📄 กระดาษ A4 (Laser/Inkjet)
— Mini Label: ~64×52mm, 3×5 = 15 ดวง/A4
— Sticker 100×50mm: 2×5 = 10 ดวง/A4

🔖 Thermal Roll Printer (เช่น Gprinter AY-AT240, Xprinter)
— ขนาดสติ๊กเกอร์: 100×50mm ต่อดวง
— Layout: QR Code ซ้าย + ข้อมูลขวา
— 1 Label = 1 หน้าพิมพ์ (ตั้งค่า Paper Size = 100×50mm)
— รองรับ 203/300/600 DPI

⚙️ การตั้งค่า Thermal Printer

1. เปิด Print Dialog → เลือก Printer = Thermal Label Printer ของคุณ
2. ตั้ง Paper Size = 100 × 50 mm (หรือ 4" × 2")
3. ตั้ง Scale = 100% (ห้าม Fit to Page)
4. ตั้ง Margins = None / Minimum
5. ปิด Headers & Footers

หากพิมพ์เล็ก/ใหญ่เกินไป ให้ลองเปลี่ยน DPI ในหน้าสร้าง Label

📦 Label FG (สินค้าสำเร็จรูป) — สำหรับลูกค้า

Label ประเภท FG ถูกปรับให้เหมาะกับการส่งออกไปขาย/ส่งลูกค้า:
— ฟิลด์ Supplier เปลี่ยนเป็น Customer โดยอัตโนมัติ
— PO No. และ Pack X/Y จะไม่แสดงบนหน้า Label (ป้องกันความสับสน)
— ข้อมูลทั้งสองยัง ฝังอยู่ใน QR/Barcode เหมือนเดิม สำหรับสแกนเข้าระบบ WMS

Label ประเภท RM (วัตถุดิบ) ยังคงใช้ Supplier / PO No. / Roll แสดงผลตามปกติ ไม่มีการเปลี่ยนแปลง

🏢 Auto-fill Supplier / Customer (NEW v6.4)

เมื่อพิมพ์หรือเลือก Part No. ที่มีอยู่ใน Master Data ระบบจะดึง ชื่อ Supplier (สำหรับ RM) หรือ ชื่อ Customer (สำหรับ FG) จาก Master Data Column E มาเติมในฟิลด์ Supplier/Customer ให้อัตโนมัติ

— ฟิลด์นี้ยังแก้ไขได้ เผื่อกรณีข้อมูลผิดหรือต้องการระบุ Supplier/Customer อื่น
— เมื่อพิมพ์ Part No. ใหม่ที่ไม่ตรงกับ Master Data ฟิลด์จะไม่ถูกเขียนทับข้อมูลที่กรอกไว้เอง
— สามารถระบุ Supplier/Customer ตอนเพิ่มสินค้าใหม่ (Local) ผ่านปุ่ม ➕ ได้เช่นกัน

📊 Data String มาตรฐาน WMS

Part | Lot | Qty | Unit | Roll | Ref | PO

ใช้ได้ทั้ง Barcode (CODE128) และ QR Code · Scanner 1D/2D สแกนได้ทั้งสองรูปแบบ
